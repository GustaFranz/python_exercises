# 28 - Avaliacao - Merge triplo e modulos src

## Demanda

Apos concluir os 27 exercicios, falta fechar duas lacunas do projeto real: o **merge triplo** em `consolidacao.py` e os **imports entre modulos** em `src/`. Esta avaliacao consolida esse entendimento.

## Conceitos revisados nesta avaliacao

- merge encadeado (simulados + provas + projetos)
- chaves de join: turma, aluno, disciplina
- formato longo antes do merge
- imports `from src.modulo import funcao`
- papel de `main.py` vs modulos em `src/`
- fluxo de dados entre leitores, consolidacao e boletins

## Instrucoes

1. Abra o projeto [automacao-dashboard-boletim-escolar](https://github.com/GustaFranz/automacao-dashboard-boletim-escolar) e deixe abertos `src/consolidacao.py`, `main.py` e pelo menos um leitor (`leitor_simulados.py`).
2. Abra `perguntas.md` e responda as **15 perguntas** por escrito (caderno ou `minhas_respostas.md`).
3. Somente apos responder, abra `gabarito.md` para conferir.
4. Se errou, revise os exercicios **10, 11, 12, 14, 15, 26** e releia os modulos citados no gabarito.

## Referencia no projeto real

- `src/consolidacao.py` — merge triplo e formula de media
- `main.py` — imports e orquestracao
- `src/leitor_simulados.py`, `src/leitor_provas.py`, `src/leitor_projetos.py`
- `src/boletins.py` — import de `consolidar_notas`
